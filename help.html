Getting Started

CodX Editor is a browser-based coding workspace for HTML, CSS, and JavaScript projects. You can create files, preview pages live, debug issues, collaborate with others, and export your project as a ZIP.

  • Welcome page: asks you to accept the terms before using the editor.
  • Launch flow: takes you into the editor after acceptance.
  • Starter experience: new users see onboarding hints, a starter page, and tutorial support.
  • Terms gate: direct access to the editor is blocked until terms are accepted.

Editing & Files

Header & More Menu

  • The main header keeps core actions visible: Collab With Friends, Fullscreen, Settings, Add Media, and Help.
  • Run, Export ZIP File, and Import ZIP File stay in the lower editor action bar.
  • The MORE dropdown now holds New, Save Project, Open Saved, Templates, Publish / Share, Zen Mode, and Fonts.
  • The dropdown opens from the header button and closes on outside click or Esc.

File Explorer

  • Create new `.html`, `.css`, `.js`, and `.env` files.
  • Rename files and keep the editor in sync.
  • Delete files and update the preview immediately.
  • Switch between files inside one project.

Import / Export

  • Export the full project as a ZIP file.
  • Name the ZIP before exporting.
  • Import ZIP projects back into the editor.
  • Drag and drop supported source files into the editor.

Saved Projects & Autosave

  • New starts a fresh HTML starter project from the MORE menu.
  • If the current project has unsaved changes, CodX asks you to save it first before starting fresh.
  • Save the current project into your browser with a custom project name.
  • Open saved projects again from the project library modal.
  • Delete saved projects with a full-screen confirmation popup so removal is harder to do by mistake.
  • Autosave keeps a draft of your current project and restores it on reload.
  • The file toolbar now shows a visible Saved or Unsaved status plus the latest autosave time.

Starter Templates

  • Open built-in starter templates from the project library.
  • Templates are shown as richer cards with icons, file counts, file types, and quick highlights.
  • Templates load multiple files at once.
  • Current built-in options include landing page, portfolio, and contact form starters.

Typing Experience

  • Tab inserts 3 spaces.
  • Undo/redo behavior is protected for editor mutations.
  • Auto-close helps with tags, quotes, and brackets.
  • Typing { in CSS, JS, <style>, and <script> immediately inserts the closing brace and places the caret between them.
  • CSS property suggestions now insert declarations that end with a semicolon, with the cursor placed before it.
  • New HTML files start clean instead of showing old welcome content.

HTML Starter Shortcut

  • Type cxstart and press Enter in an empty HTML file.
  • CodX inserts a starter HTML structure for you.
  • An editor watermark reminds you about this when the HTML editor is empty.

Preview & Console

Live Preview

  • Preview HTML, CSS, and JS together inside the iframe.
  • Auto-Run refreshes preview while you work.
  • Manual Run is available if Auto-Run is off.
  • The preview header shows the current page title from the HTML <title> tag.

Linked Files

  • HTML pages can link to other project files in preview.
  • Buttons and forms that navigate to project pages are supported in preview.
  • <a href="#"> and missing HTML links stay inside the iframe flow and open 404-for-preview.html.
  • Missing preview pages go to 404-for-preview.html.

Media Support

  • Add images, video, and audio files into the project.
  • Media sources inside preview are rewritten so controls actually work.
  • Linked <source> tags are supported.

Console Output

  • See logs, warnings, and runtime errors.
  • Clear console output any time.
  • Unhandled promise rejections and iframe runtime issues are captured.

Publish & Share

  • Publish the current project and generate a shareable public link.
  • The published page renders your current HTML project with linked CSS and JS files.
  • The link is shown after publishing and can be copied for sharing.
  • Published projects are now stored server-side so shared links survive server restarts better.

Settings & Fonts

Editor Settings

  • Change editor background color.
  • Change editor text size.
  • Preview settings before applying them.
  • Reset everything to defaults.

Font Picker

  • Pick from many built-in fonts.
  • Search font names inside the font modal.
  • Distinct font choices were added to avoid too many lookalikes.

Google Fonts Embed

  • Paste a Google Fonts embed URL or snippet.
  • CodX uses @import to load the font.
  • When a Google Fonts embed is active, the normal font dropdown is disabled.
  • Pasting a new Google Fonts link now replaces the previous one properly instead of sticking to the old font.
  • Your applied Google Fonts choice now stays active after refresh.
  • The settings preview updates while you paste so you can see the font change before applying.
  • This feature needs internet access to fetch the font.

Zen Mode

  • Hides most UI so you can focus only on typing.
  • Has a visible exit button inside the editor.
  • Escape also exits Zen Mode.

Errors, Highlighting & Suggestions

Error Detection

  • Checks HTML, CSS, JS, and inline script issues.
  • Shows file error badges in the explorer.
  • If error count is above 99, the badge shows 99+.

Jump To Error

  • Click the error badge to jump to the first error in that file.
  • The editor moves the caret to the error location when possible.
  • HTML tag errors are highlighted in red where the problem exists.

Inline Highlighting

  • Error lines receive a red background highlight.
  • Specific error ranges are underlined when location data is available.
  • Closing HTML tags now receive syntax colors too.
  • CSS inside <style> and JavaScript inside <script> now use their own syntax highlighters inside HTML files.

Smart Suggestions

  • HTML tag suggestions appear while typing.
  • Typing plain HTML words can suggest tags even before you type <.
  • Attribute suggestions work inside tags.
  • The lorem snippet suggestion inserts lorem ipsum placeholder text.
  • Inline style="" suggestions support CSS properties and values.
  • CSS and JS suggestion flows are also supported.
  • Property-based CSS values are suggested too, including richer font-family stacks and common values for typography, sizing, spacing, layout, and effects.
  • Typing a number in CSS value positions can suggest the matching px version.
  • Color-related properties such as color and background-color can show value suggestions with a real color swatch box beside the color text.

Collaboration Basics

Sessions

  • Create a collaboration session and share a link.
  • Join by link with your name and chosen color.
  • Refresh while collabing starts fresh instead of silently rejoining from URL.

Chat

  • Group chat and private chat are built into the collab modal.
  • Messages show the sender color they picked.
  • If someone sends you a private message, CodX shows a notification with the sender name in bold.
  • Group chat can be cleared by moderators with permission.

Live Presence

  • See collaborator cursors in their chosen color.
  • See typing highlights where others are typing.
  • Typing and cursor state clear properly on disconnect or session reset.

Session Safety

  • Invalid collab links go to the 404 page.
  • Kicked users see a notice modal and are reset to a fresh state.
  • Banned users cannot rejoin that same collaboration session from the same device until unbanned.
  • If host leaves and a co-host exists, co-host is promoted first.

Built-In Assets

Built-In Logo Asset

  • cx.png is built into CodX Editor.
  • You can use it directly in your code without importing or downloading it first.
  • This works in the editor preview and in published project pages.

Host, Co-Host & Participant Roles

Host

  • Full room control.
  • Can manage participants, change roles, and use all room-wide controls.
  • Can transfer host to another user with confirmation.

Co-Host

  • Limited moderator/helper role.
  • Can kick normal participants, mute chat, freeze editing, manage allowed files, mark priority, follow users, bring everyone to file, pin team file, and clear group chat.
  • FOLLOW USER now follows both the user's current file and their live cursor position inside the editor in real time.
  • Cannot lock room, end session, transfer host, or change room ownership controls.

Participant

  • Can edit, preview, chat, and collaborate normally.
  • May be limited by file access, room settings, mute, or frozen editing.

Participant Tools

  • Host and co-host can open MORE on participants.
  • There you may see message, copy name, copy role, copy color, kick, follow user, file access, and detail tools depending on role.
  • If you use FOLLOW USER, your editor view will stay with that person's file and scroll with their live cursor while they move around.

Room & Group Controls

These tools live in the collaboration modal and are mainly for the host. Some limited team tools are available to co-hosts.

Group Controls

  • Lock room
  • Read-only for all
  • Disable chat for all
  • Disable run for participants
  • Disable console for participants
  • Bring everyone to file
  • Pin team file
  • Clear group chat

Broadcast & Focus

  • Announcement bar
  • Announcement popup on screen
  • Group highlight mode
  • Team focus indicators in the file list

Session Management

  • Session timer
  • Pause collaboration
  • Quiet mode
  • Save session snapshot
  • Regenerate invite link
  • End session for everyone

Join Approval

  • Approve new joins can be turned on.
  • Pending join requests show inside the group controls view.
  • Host can approve or reject waiting users.

Shortcuts

Ctrl/Cmd + SExport project as ZIP
Ctrl/Cmd + EnterRun preview manually
Ctrl/Cmd + QCreate a new file
Ctrl/Cmd + Shift + CToggle console panel
EscExit Zen Mode and close supported overlays
cxstart + EnterInsert the HTML starter setup in an empty HTML file

Hidden Developer Tools

CodX includes hidden internal tools for testing and quick inspection of the current editor state.

  • Shortcut: press Ctrl/Cmd + C, then X.
  • Purpose: internal testing and quick inspection of files, participants, permissions, errors, and preview state.
  • Commands: help, state, files, participants, permissions, errors, preview, clear, close.
